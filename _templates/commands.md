# Output Format (输出要求)
请严格按照以下结构输出文档：

## 1. 文档概述
-   **项目背景**：用专业的语言重述我提供的背景。
-   **用户价值**：这个功能为业务带来了什么价值？
-   **适用范围**：哪些角色会用到这个功能？

## 2. 业务流程图 (Mermaid)
-   请根据我的描述，梳理出**主流程**和**异常流程**。
-   请用 Mermaid 代码绘制泳道图 (Sequence Diagram 或 Flowchart)，区分不同角色（如：采购员、仓库管理员、系统）。

## 3. 详细功能需求 (Functional Requirements)
请把功能拆解为具体的“功能点”。对于每个功能点，请包含：
-   **前置条件**：进入该功能/页面的条件。
-   **页面元素与交互**：页面上有哪些字段、按钮？支持什么操作（筛选、排序、导出）？
-   **逻辑规则（核心）**：这是最重要的部分。请详细列出后端校验逻辑、状态流转逻辑、数据计算公式。
    -   *Example:* “点击提交时，系统需校验‘退货数量 ≤ 原单可退数量’，若校验失败，提示错误信息...”
-   **后置操作**：操作成功后，数据如何变化？页面如何跳转？

## 4. 数据字典与字段定义 (Data Requirement)
-   请以表格形式列出该功能涉及的核心字段（如：表单输入项）。
-   表格包含：**字段名称、字段类型、是否必填、默认值、数据来源/校验规则**。

## 5. 异常流程与边界情况 (Edge Cases)
-   请帮我补充我想不到的异常情况。例如：网络中断、数据并发冲突、关联单据状态已变更等，系统应该如何处理？

## 6. 埋点与统计需求 (可选)
-   如果有数据统计需求，请列出。
